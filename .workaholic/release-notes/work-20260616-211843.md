# gdrive-ftp — fix `put` into a folder; binary-transfer docs

PR: https://github.com/qmu/gdrive-ftp/pull/2

## Fixed

- **`put` into an existing folder** no longer creates a same-named file in the
  parent (which shadowed the folder, wedged the path with `ambiguous name`
  errors, and silently overwrote on repeat uploads). `put <file> "/My Drive/Work"`
  now uploads **into** `Work` under the local filename. Use `put <file>
  "/My Drive/Work/newname.pdf"` to upload under a different name; a trailing slash
  still means "into this directory".

## Docs

- README documents that transfers are **byte-for-byte binary** (no base64 or
  conversion): PDFs, Office files (DOCX/XLSX/PPTX), ZIPs, and any binary upload
  and download identically at any size. Downloads are length-checked against
  Drive's reported size.

## Notes

A path already broken by the previous behavior (a folder and a same-named file)
still needs the Drive web UI to clean up, since the CLI matches by name.
