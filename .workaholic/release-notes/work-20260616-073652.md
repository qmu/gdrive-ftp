# gdrive-ftp — remote OAuth, Shared Drives, completion, and a public plugin

PR: https://github.com/qmu/gdrive-ftp/pull/1

## Added

- **Shared Drives**: list and navigate every Shared Drive you can access, not
  just My Drive.
- **Virtual root**: the shell opens at `/` listing `My Drive/` and each Shared
  Drive; the drive name is the first path component (e.g. `/My Drive/Work`).
- **Tab completion**: sftp-style completion inside the shell, plus a zsh
  prompt-completion script (`source <(gdrive-ftp completion zsh)`) for
  `gdrive-ftp <cmd> <Tab>` — verbs, live remote Drive paths, and local paths.
- **`auth` subcommand**: `gdrive-ftp auth` runs the OAuth consent flow and exits.
- **Agent plugin**: installable on Claude Code and OpenAI Codex (and the
  cross-agent skills CLI), shipping a skill that teaches agents to drive the CLI.
  MIT licensed.

## Changed

- **OAuth is now terminal-based by default** and works over SSH: press `c` to
  copy the consent URL to your local clipboard (OSC 52, tmux-aware), then paste
  the `127.0.0.1` redirect URL back. The old loopback flow and `-manual` flag are
  gone. CSRF `state` is validated and authorization denials are surfaced.
- A disabled Google Drive API now produces a clear, actionable message (with the
  exact enable URL) instead of a raw API error.

## Upgrade notes

- Paths now begin with the drive name — use `/My Drive/...` (quote names with
  spaces). The `gdrive-ftp` binary is installed separately from the plugin; run
  `gdrive-ftp auth` once before agent use.
- Requires **full Drive access**: `put` can overwrite same-named files and `rm`
  trashes (reversible). Keep `credentials.json`/`token.json` private.
