# Add `id:` addressing, `-json` output, and a `find` command

## Summary

This release makes gdrive-ftp far more usable by scripts and AI agents: address any item by its Google Drive ID, get machine-readable JSON from every command, and search Drive by name.

## Key Changes

- **`id:<DriveID>` addressing** across `get`/`put`/`rm`/`ls`/`cd`/`mkdir` — target a file or folder directly by ID, the unambiguous way to clean up wedged folder+file name collisions.
- **Global `-json` flag** — every command emits compact JSON (results on stdout, errors on stderr with a non-zero exit), so output no longer needs scraping.
- **`find <pattern> [dir]` command** — search the current drive by name (Drive's native `name contains`), printing full paths you can then act on by `id:`.

## Changes

### Added
- `id:<DriveID>` target form for all path-taking commands, plus a `GetByID` client lookup.
- `-json` machine-readable output mode with a stable stdout/stderr/exit contract.
- `find` command for name search, scoped to the current drive, reusing the JSON output.
