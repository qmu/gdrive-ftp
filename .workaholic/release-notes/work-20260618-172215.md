# Add an audit log of Drive mutations and a `log` browser

## Summary

gdrive-ftp now keeps a recoverable record of what it changes. Every `put`, `rm`,
and `mkdir` is logged, and a new `gdrive-ftp log` command lets you review that
history — useful when an AI agent driving the CLI touches files you didn't intend.

## Key Changes

- **Audit log** — every mutating operation is appended to
  `~/.config/gdrive-ftp/audit.jsonl` (append-only JSON Lines, `0600`, size-rotated).
  Each record captures what changed, including the before/after size of an
  overwriting `put`. It never stores credentials or file contents. `-no-log`
  disables it.
- **`gdrive-ftp log`** — a read-only, `tig`-like terminal browser (`j`/`k` to move,
  `Enter` for detail, `q` to quit), built with no new dependencies.
- **Scriptable** — piped or with `-json`, `gdrive-ftp log` prints the entries as
  plain rows or a JSON array, so scripts and AI agents read the same history.

## Changes

### Added
- Audit logging of `put`/`rm`/`mkdir` to a rotated JSONL file under the config dir.
- A `gdrive-ftp log` subcommand: interactive browser plus plain/`-json` output.
